Guia para iniciantes

Guia de deploy do JurisprudenciaIA MCP

Um roteiro guiado para preparar os secrets, publicar o Worker e conectar o MCP no Claude.ai, ChatGPT ou Codex com segurança e previsibilidade.

Para quem este guia foi escrito

Para quem quer seguir um roteiro prático, sem precisar entender todos os detalhes técnicos antes de começar.

  • Tempo estimado: 30 a 45 minutos.
  • Resultado final: um Worker publicado e testado.
  • Você copia um comando por vez.
  • Você usa apenas valores fictícios neste documento.
  • Você coloca os valores reais somente na sua conta Cloudflare.
1. Preparar

Instalar dependências e confirmar que o projeto abre sem erro.

2. Proteger

Criar chaves fora do repositório, sem colar valores reais em arquivos públicos.

3. Publicar

Enviar o Worker para a Cloudflare e guardar a URL terminada em /mcp.

4. Conectar

Usar a URL no Claude.ai, ChatGPT ou Codex e testar antes do uso profissional.

    Fase 1 - Preparar sem pressa

    Nesta fase você confirma que está na pasta certa, instala os arquivos necessários e roda a verificação local.

    Etapas 1 a 5 Tempo médio: 10 minutos Você ainda não mexe em chaves
    1

    Entenda o que você vai montar

    O JurisprudenciaIA MCP será publicado como um Worker na sua conta Cloudflare. Depois disso, Claude.ai, ChatGPT ou Codex conversam com esse Worker usando uma URL de MCP.

    Em linguagem simples O Worker funciona como uma ponte: o assistente recebe sua pergunta, chama o MCP e devolve a pesquisa jurídica organizada.
    Worker

    É o serviço publicado na Cloudflare.

    Secret

    É uma chave sensível, como uma senha.

    OAuth

    É a autorização usada por Claude.ai e ChatGPT.

    Você precisa guardar

    A URL final do Worker, sempre terminando com /mcp.

    Você não deve publicar

    Secrets, tokens, IDs pessoais, domínios privados ou anotações internas.

    2

    Separe o que será necessário

    Antes de abrir o terminal, confirme se você tem os itens abaixo. Isso evita parar no meio do processo.

    1. Uma conta Cloudflare com Workers habilitado.
    2. Node.js 22 ou superior instalado.
    3. A pasta deste projeto aberta no computador.
    4. Um navegador para entrar na Cloudflare quando o Wrangler pedir login.
    Não sabe se o Node está instalado? Abra o terminal e rode o comando da próxima etapa. Se aparecer uma versão, você pode seguir.
    3

    Abra o terminal na pasta do projeto

    No Windows, use PowerShell ou Terminal. O ponto importante é estar dentro da pasta onde ficam o package.json e o wrangler.toml.

    Conferir Node e pasta
    node --version
npm --version
dir package.json
dir wrangler.toml
    Resultado esperado Os dois primeiros comandos mostram versões. Os dois últimos encontram os arquivos do projeto.
    4

    Instale os arquivos necessários

    Este comando baixa as dependências do projeto. Ele pode demorar alguns minutos na primeira vez.

    Instalação
    npm install
    Se aparecer uma pergunta do Windows Permita a execução somente se você estiver na pasta correta do projeto e baixou o código de uma fonte confiável.
    5

    Faça a verificação local

    Antes de publicar, peça ao projeto para conferir tipos, testes, auditoria e build. É melhor encontrar problemas agora do que depois de conectar o cliente.

    Verificação completa
    npm run verify
    Resultado esperado O comando termina sem erro. Se falhar, resolva a mensagem antes de continuar.

    Fase 2 - Criar e colocar as chaves

    Aqui você cria os valores sensíveis e coloca esses valores diretamente na Cloudflare. Eles não entram em arquivos públicos.

    Etapas 6 a 14 Tempo médio: 15 a 25 minutos Atenção máxima com copiar e colar
    6

    Crie uma folha temporária para as chaves

    Você vai gerar alguns valores secretos. Use uma anotação temporária só para copiar e colar durante a configuração.

    1. Abra um bloco de notas local.
    2. Escreva os nomes dos campos da tabela abaixo.
    3. Preencha os valores gerados nas próximas etapas.
    4. Apague essa anotação quando terminar os testes.
    CampoPara que serve
    MCP_OAUTH_CLIENT_IDIdentifica o cliente OAuth usado por Claude.ai ou ChatGPT.
    MCP_OAUTH_CLIENT_SECRETSenha secreta do cliente OAuth.
    MCP_ACCESS_TOKEN_SECRETAssina os tokens emitidos pelo Worker.
    MCP_OAUTH_REDIRECT_URISLista de endereços de retorno aceitos no OAuth.
    MCP_BEARER_TOKENToken usado pelo Codex quando a conexão for HTTP com streaming.
    Importante Esses valores são como senhas. Não cole valores reais em arquivos do repositório.
    7

    Crie as chaves secretas

    Copie o bloco abaixo no terminal. Ele imprime valores aleatórios para você usar como chaves secretas.

    Criar chaves
    node -e "const crypto=require('node:crypto'); console.log('MCP_OAUTH_CLIENT_ID=jurisprudenciaia-mcp-client'); console.log('MCP_OAUTH_CLIENT_SECRET='+crypto.randomBytes(32).toString('hex')); console.log('MCP_ACCESS_TOKEN_SECRET='+crypto.randomBytes(32).toString('hex')); console.log('MCP_BEARER_TOKEN='+crypto.randomBytes(32).toString('hex'));"
    1. Copie o valor que aparece depois de MCP_OAUTH_CLIENT_SECRET=.
    2. Copie o valor que aparece depois de MCP_ACCESS_TOKEN_SECRET=.
    3. Copie o valor que aparece depois de MCP_BEARER_TOKEN=, se for usar Codex.
    Sobre o redirect URI Use apenas URLs de retorno oficiais do cliente que você estiver conectando. Se ainda não sabe qual usar, configure primeiro o Worker e volte nesta etapa quando o cliente mostrar o endereço de retorno.
    8

    Entre na Cloudflare pelo terminal

    O Wrangler é a ferramenta que publica o Worker. O comando abaixo abre o navegador para você autorizar a conta Cloudflare.

    Login do Wrangler
    npx wrangler login
    Resultado esperado O navegador abre, você confirma o acesso e o terminal informa que o login foi concluído.
    9

    Confira o nome do Worker

    O arquivo wrangler.toml já traz uma configuração genérica. Nesta etapa você só confere se o arquivo existe e se o nome do Worker faz sentido para a sua conta.

    Abrir configuração
    notepad wrangler.toml
    Não coloque chaves aqui O wrangler.toml pode ter nome e compatibilidade do Worker. Tokens e chaves entram pelo comando wrangler secret put, não neste arquivo.
    10

    Coloque o ID do cliente OAuth na Cloudflare

    Copie o comando, pressione Enter e cole o valor quando o terminal pedir. Para o ID, você pode usar o valor genérico abaixo.

    Chave 1 de 5
    npx wrangler secret put MCP_OAUTH_CLIENT_ID
    Valor sugerido quando o terminal pedir jurisprudenciaia-mcp-client
    11

    Coloque a chave OAuth na Cloudflare

    Agora grave o valor gerado para MCP_OAUTH_CLIENT_SECRET. Cole somente o valor depois do sinal de igual.

    Chave 2 de 5
    npx wrangler secret put MCP_OAUTH_CLIENT_SECRET
    Exemplo do que não fazer Não cole MCP_OAUTH_CLIENT_SECRET=... inteiro. Cole apenas o valor secreto que veio depois do =.
    12

    Coloque a chave de assinatura dos tokens

    Este valor protege os tokens emitidos pelo Worker. Use o valor gerado para MCP_ACCESS_TOKEN_SECRET.

    Chave 3 de 5
    npx wrangler secret put MCP_ACCESS_TOKEN_SECRET
    Conferência simples Se o terminal responder que o secret foi criado ou atualizado, siga para a próxima etapa.
    13

    Coloque os endereços de retorno do OAuth

    Claude.ai e ChatGPT usam URLs de retorno no OAuth. Configure somente os endereços informados pelo cliente que você está conectando.

    Chave 4 de 5
    npx wrangler secret put MCP_OAUTH_REDIRECT_URIS
    Formato esperado Se houver mais de uma URL, separe por vírgula, sem espaço. Use apenas URLs públicas oficiais do cliente.
    14

    Coloque o token do Codex, se for usar Codex

    O Codex pode conectar por HTTP com streaming usando Bearer token. Se você não for usar Codex agora, pode pular esta etapa e voltar depois.

    Chave 5 de 5
    npx wrangler secret put MCP_BEARER_TOKEN
    Guarde com cuidado O valor local do Codex precisa ser igual ao valor colocado no Worker.

    Fase 3 - Publicar

    Com as chaves já colocadas na Cloudflare, você publica o Worker e guarda a URL pública terminada em /mcp.

    Etapa 15 Tempo médio: 5 minutos Resultado: URL do Worker
    15

    Publique o Worker

    Com as chaves colocadas, publique o Worker. Ao final, o terminal mostra a URL pública do serviço.

    Publicação
    npx wrangler deploy
    1. Copie a URL que o Wrangler mostrar.
    2. Acrescente /mcp no final, se a URL ainda não tiver esse caminho.
    3. Guarde a URL no formato https://<sua-url-do-worker>/mcp.

    Fase 4 - Conectar e testar

    Escolha o cliente que você quer usar primeiro. Claude.ai e ChatGPT seguem o caminho OAuth; Codex segue o caminho por token.

    Etapas 16 a 18 Tempo médio: 10 minutos Faça teste antes de usar em caso real
    16

    Conecte Claude.ai ou ChatGPT

    Para Claude.ai e ChatGPT, use a opção de conector/app personalizado com OAuth. O nome exato da tela pode mudar, mas a lógica é a mesma.

    1. Abra a área de conectores ou apps personalizados.
    2. Informe a URL MCP: https://<sua-url-do-worker>/mcp.
    3. Use o client ID configurado no Worker.
    4. Use o client secret configurado no Worker.
    5. Quando o cliente mostrar uma URL de retorno, confirme se ela está em MCP_OAUTH_REDIRECT_URIS.
    6. Salve e faça a autorização.
    Sinal de que funcionou O cliente consegue listar ferramentas MCP ou conclui a autorização sem erro.
    17

    Conecte o Codex, se for usar

    No Codex, configure o MCP remoto por HTTP com streaming. A URL é a mesma do Worker e o token local precisa ser igual ao MCP_BEARER_TOKEN colocado na Cloudflare.

    URL do servidor

    https://<sua-url-do-worker>/mcp

    Token

    Use o mesmo valor salvo como MCP_BEARER_TOKEN.

    Referência específica Para detalhes do Codex, consulte também docs/codex.md.
    18

    Teste antes de usar em trabalho real

    Os testes abaixo confirmam se o Worker responde e se as ferramentas MCP aparecem. Substitua a URL de exemplo pela URL do seu Worker.

    Teste de conexão
    npm run check:chatgpt-oauth -- https://<sua-url-do-worker>/mcp
npm run check:codex-http -- https://<sua-url-do-worker>/mcp
    Resultado esperado O terminal lista ferramentas como consultar_jurisprudenciaia, buscar_precedentes e analisar_tese_juridica.

    Se aparecer erro, comece por aqui

    Erros de conexão costumam ser causados por URL incompleta, secret ausente ou token diferente entre Cloudflare e cliente.

    O cliente diz HTTP 401 ou HTTP 403

    Confira se o token ou chave usada no cliente é o mesmo valor colocado no Worker. Se estiver em dúvida, crie um novo valor, coloque novamente com wrangler secret put e atualize o cliente.

    O teste não encontra ferramentas MCP

    Confirme se a URL termina com /mcp. Também confirme se o deploy terminou sem erro e se você está testando a URL do Worker correto.

    O OAuth volta para uma página de erro

    Revise MCP_OAUTH_REDIRECT_URIS. O endereço de retorno informado pelo cliente precisa estar exatamente na lista colocada no Worker.

    O terminal diz que não reconhece npm, node ou npx

    Instale ou reinstale o Node.js 22 ou superior. Depois feche e abra novamente o terminal antes de repetir os comandos.

    Depois que tudo funcionar

    Faça uma limpeza final para reduzir risco de vazamento e facilitar manutenção futura.

    • Apague a anotação temporária que tinha chaves.
    • Guarde tokens apenas em local seguro ou no ambiente onde forem necessários.
    • Não publique prints com URL privada, tokens, IDs ou telas de configuração sensíveis.
    • Use docs/deployment.md como referência curta para manutenção.
    • Revise sempre a fonte oficial antes de citar jurisprudência em peça, parecer ou minuta.
