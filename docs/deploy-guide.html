Guia para iniciantes

Guia de deploy do JurisprudenciaIA MCP

Um roteiro guiado para preparar os secrets, publicar o Worker e conectar o MCP no Claude.ai, ChatGPT ou Codex com segurança e previsibilidade.

Antes de começar

Você vai copiar valores secretos durante o processo. Eles são como senhas: não coloque em README, issue, print, conversa pública, URL ou arquivo commitado. Use apenas os campos da Cloudflare e as variáveis locais do seu computador.

Seu progresso

Marque cada etapa concluída. O navegador guarda esta lista neste computador.

0 de 9 etapas concluídas.

    1

    Prepare a pasta do projeto

    Abra o terminal na pasta do repositório. Se você não usa terminal com frequência, faça isso com calma: copie um comando por vez, cole, pressione Enter e espere terminar.

    1. Instale o Node.js 22 ou superior.
    2. Instale as dependências do projeto.
    3. Rode a verificação local antes de publicar.
    Terminal
    npm install
npm run verify
    2

    Crie uma folha temporária de anotação

    Use um arquivo local temporário, fora do Git, só para copiar e colar durante a configuração. Apague esse arquivo quando tudo estiver funcionando.

    Modelo de rascunho
    MCP_OAUTH_CLIENT_ID=jurisprudenciaia-mcp-client
MCP_OAUTH_CLIENT_SECRET=<cole aqui depois de gerar>
MCP_ACCESS_TOKEN_SECRET=<cole aqui depois de gerar>
MCP_OAUTH_REDIRECT_URIS=<cole as URLs autorizadas>
MCP_BEARER_TOKEN=<opcional para Codex>
URL_WORKER=https://<sua-url-do-worker>
    Importante

    Essa folha não vai para o repositório. Ela é só um apoio temporário para evitar erro de digitação.

    3

    Gere os valores secretos

    Os dois primeiros valores protegem o OAuth usado por Claude.ai e ChatGPT. O terceiro é usado apenas se você for conectar o Codex por Bearer token.

    PowerShell ou terminal
    node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
    1. Cole o primeiro resultado em MCP_OAUTH_CLIENT_SECRET.
    2. Cole o segundo resultado em MCP_ACCESS_TOKEN_SECRET.
    3. Se for usar Codex, cole o terceiro resultado em MCP_BEARER_TOKEN.
    4

    Entre na sua conta Cloudflare

    O Wrangler é a ferramenta que publica o Worker. O comando abaixo abre o login da Cloudflare no navegador.

    Login
    npx wrangler login
    5

    Grave os secrets no Worker

    Rode um comando por vez. Quando o Wrangler pedir o valor, cole exatamente o valor correspondente da sua folha temporária.

    Secrets obrigatórios e opcional do Codex
    npx wrangler secret put MCP_OAUTH_CLIENT_ID
npx wrangler secret put MCP_OAUTH_CLIENT_SECRET
npx wrangler secret put MCP_ACCESS_TOKEN_SECRET
npx wrangler secret put MCP_OAUTH_REDIRECT_URIS
npx wrangler secret put MCP_BEARER_TOKEN
    O que colocar em MCP_OAUTH_REDIRECT_URIS

    Use as URLs de retorno exigidas pelo cliente que você vai conectar. Se ainda não souber, deixe esta etapa para quando o Claude.ai ou ChatGPT mostrar a URL de retorno na tela de configuração.

    6

    Publique o Worker

    O repositório público não traz domínio customizado. A primeira publicação pode usar a URL padrão da Cloudflare. Depois, se quiser, você configura um domínio próprio na sua conta.

    Publicação
    npx wrangler deploy
    1. Ao terminar, copie a URL HTTPS mostrada pelo Wrangler.
    2. Na sua folha temporária, preencha URL_WORKER=https://<sua-url-do-worker>.
    3. A URL do MCP será essa URL com /mcp no final.
    7

    Conecte Claude.ai ou ChatGPT

    Para Claude.ai e ChatGPT, use OAuth. Copie os nomes e valores com atenção; um caractere diferente impede a conexão.

    URL do MCP

    https://<sua-url-do-worker>/mcp

    Client ID

    jurisprudenciaia-mcp-client

    Client Secret

    Use o valor de MCP_OAUTH_CLIENT_SECRET.

    Autenticação

    Use client_secret_post quando o cliente pedir o método do endpoint de token.

    8

    Conecte o Codex

    No Codex, o segredo não é colado na tela do MCP. Você grava uma variável local chamada MCP_BEARER_TOKEN e informa apenas o nome dessa variável.

    Windows
    [Environment]::SetEnvironmentVariable("MCP_BEARER_TOKEN", "<mesmo valor do Worker>", "User")
    macOS
    launchctl setenv MCP_BEARER_TOKEN "<mesmo valor do Worker>"
    Na tela do Codex

    Tipo: HTTP com streaming. URL: https://<sua-url-do-worker>/mcp. Variável de token do portador: MCP_BEARER_TOKEN.

    9

    Teste antes de usar em trabalho real

    O teste deve listar as ferramentas MCP. Se falhar, leia a mensagem: normalmente é URL errada, secret ausente ou token local diferente do token do Worker.

    Teste de conexão
    npm run check:chatgpt-oauth -- https://<sua-url-do-worker>/mcp
npm run check:codex-http -- https://<sua-url-do-worker>/mcp
    1. Se aparecer HTTP 401 ou HTTP 403, confira os secrets.
    2. Se aparecer timeout, tente novamente e confirme se a URL termina com /mcp.
    3. Depois que funcionar, apague a folha temporária com os valores.

    Quando terminar

    Você terá um Worker próprio, conectado ao cliente escolhido e pronto para consultar o JurisprudenciaIA via MCP.

    • Guarde secrets somente no Worker e no ambiente local necessário.
    • Revise os resultados jurídicos antes de citar qualquer precedente.
    • Use docs/deployment.md como referência curta para manutenção.
