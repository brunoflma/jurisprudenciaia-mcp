Guia de deploy do JurisprudenciaIA MCP

Um roteiro guiado para preparar os secrets, publicar o Worker e conectar o MCP no Claude.ai, ChatGPT ou Codex com segurança e previsibilidade.

1 Preparar

Separe os valores temporários e gere os secrets do MCP.

2 Publicar

Configure os Worker secrets e publique pelo controlador de deploy escolhido.

3 Conectar

Teste o Worker e registre o conector no Claude.ai, ChatGPT ou Codex.

  • Quando o guia mostrar texto entre <assim>, substitua pelo valor correspondente.
  • Quando um campo pedir um secret, cole somente o valor, sem aspas e sem espaços extras.
  • Conclua a confirmação de cada etapa antes de seguir para a próxima.
0 de 9 etapas concluídas
Os marcadores ficam salvos apenas neste navegador.

Antes de começar

Você precisa ter aberto

  • GitHub, já logado na conta que possui o repositório.
  • Cloudflare Dashboard, já logado na conta que vai hospedar o Worker.
  • Claude.ai, ChatGPT ou Codex, conforme o cliente onde o conector será usado.
  • Bloco de Notas, para guardar valores temporariamente durante o deploy.

Mantenha fora do repositório

  • Tokens da Cloudflare.
  • Secrets do MCP.
  • Anotações internas de desenvolvimento.
  • Arquivos .env com valores reais.
1

Crie sua folha de anotação temporária

Esta folha serve apenas para copiar e colar durante o processo.

  1. Abra o aplicativo Bloco de Notas.
  2. Copie o modelo abaixo.
  3. Cole no Bloco de Notas.
  4. Preencha cada linha somente quando o guia pedir.
Modelo para colar no Bloco de Notas 
CLOUDFLARE_ACCOUNT_ID=
CLOUDFLARE_API_TOKEN=
MCP_OAUTH_CLIENT_ID=jurisprudenciaia-mcp-client
MCP_OAUTH_CLIENT_SECRET=
MCP_ACCESS_TOKEN_SECRET=
MCP_BEARER_TOKEN=
URL_WORKER=https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev
URL_MCP=https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp

Não salve esse arquivo dentro da pasta do projeto. Quando terminar e tudo estiver funcionando, apague a anotação temporária.

2

Gere os secrets do MCP

Esses valores protegem OAuth no Claude/ChatGPT e Bearer no Codex.

  1. Abra o terminal do seu sistema: PowerShell no Windows ou Terminal no macOS.
  2. Copie o primeiro comando abaixo, cole no terminal e aperte Enter.
  3. Copie o resultado e cole na linha MCP_OAUTH_CLIENT_SECRET= da sua anotação.
  4. Repita com o segundo comando e cole o resultado em MCP_ACCESS_TOKEN_SECRET=.
  5. Se for usar Codex HTTP, repita com o terceiro comando e cole em MCP_BEARER_TOKEN=.
Primeiro comando: Client Secret 
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
Segundo comando: Access Token Secret 
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
Terceiro comando: Bearer Token para Codex 
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"

Resultado correto: cada comando mostra uma linha longa com letras e números. Se aparecer node não é reconhecido no Windows ou command not found: node no macOS, o Node.js não está instalado nesse computador.

3

Crie o token de deploy no Cloudflare

Esse token só é necessário para publicação manual via Wrangler fora de uma integração de build do provedor.

  1. Entre em dash.cloudflare.com.
  2. Clique no ícone de perfil ou em Manage Account.
  3. Entre em Account API Tokens.
  4. Clique em Create Token.
  5. Escolha o modelo Edit Cloudflare Workers. Se não aparecer, use Custom token com permissão de editar Workers.
  6. Em Token name, digite exatamente jurisprudenciaia-mcp.
  7. Em Account Resources, selecione Include e escolha a conta Cloudflare correta.
  8. Se aparecer Zone Resources, deixe restrito ao menor escopo possível. Para URL workers.dev, não precisa permissão de DNS.
  9. Clique em Continue to summary.
  10. Clique em Create Token.
  11. Copie o token mostrado e cole na linha CLOUDFLARE_API_TOKEN= da anotação.

O Cloudflare mostra esse token uma única vez. Se fechar a página sem copiar, crie outro token.

4

Copie o Account ID do Cloudflare

O Account ID identifica onde o Worker será publicado.

  1. No Cloudflare Dashboard, abra a mesma conta escolhida no token.
  2. Entre em Compute.
  3. Entre em Workers & Pages.
  4. Procure o texto Account ID na página ou na lateral direita.
  5. Copie o valor e cole na linha CLOUDFLARE_ACCOUNT_ID= da anotação.

Resultado correto: o Account ID é uma sequência longa de letras e números. Não use o nome da conta.

5

Cadastre os secrets no Worker

Crie um Worker secret por vez. O nome precisa ser idêntico ao da tabela.

  1. No Cloudflare, abra o Worker jurisprudenciaia-mcp.
  2. Entre em Settings e depois em Variables.
  3. Clique para adicionar um novo secret.
  4. Preencha Name e Value usando a tabela abaixo.
  5. Salve o secret.
  6. Repita até cadastrar os quatro secrets obrigatórios.
Name Secret
MCP_OAUTH_CLIENT_ID jurisprudenciaia-mcp-client
MCP_OAUTH_CLIENT_SECRET Primeiro valor gerado no passo 2.
MCP_ACCESS_TOKEN_SECRET Segundo valor gerado no passo 2.
MCP_OAUTH_REDIRECT_URIS URLs de redirecionamento permitidas, separadas por vírgula.

Resultado correto: a tela de variáveis do Worker mostra os quatro nomes obrigatórios. O Cloudflare não mostra os valores depois de salvar.

Se for usar Codex HTTP, crie também o secret opcional MCP_BEARER_TOKEN diretamente no Worker. GitHub Actions não sincroniza secrets nem publica produção.

6

Publique pelo controlador escolhido

O GitHub valida o projeto; o controlador de deploy publica no Cloudflare Workers.

  1. No Cloudflare, abra Workers & Pages e selecione o projeto conectado ao repositório de origem autorizado.
  2. Confirme que a branch de produção configurada está correta.
  3. Confirme que o build usa npm run verify.
  4. Confirme que o deploy usa npx wrangler deploy.
  5. Dispare o build no controlador escolhido ou aguarde o build do commit de produção.
  6. Espere o build e o deploy ficarem verdes.
  7. Procure a URL publicada, parecida com https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev.
  8. Copie essa URL e preencha as linhas URL_WORKER= e URL_MCP= na anotação.
Formato correto das URLs 
URL_WORKER=https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev
URL_MCP=https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp

A URL workers.dev funciona para o MCP. Porém o Claude.ai pode buscar o ícone pelo Google Favicon usando apenas http://<seu-subdominio>.workers.dev, sem o nome do Worker. Para que o ícone apareça de forma confiável na lista de conectores, configure um domínio customizado no Cloudflare Workers, como https://mcp.<seu-dominio>/mcp, e use essa URL no Claude.ai.

Se for usar Codex HTTP, confira se o Worker secret opcional MCP_BEARER_TOKEN existe antes de publicar. Esse mesmo valor deve existir no ambiente local do Codex.

Se algum secret obrigatório estiver ausente, o deploy fica vermelho. Volte ao passo 5; o Worker não foi publicado nesse caso.

7

Teste se o Worker está online

Este teste confirma que a URL publicada responde antes de configurar o cliente MCP.

  1. Pegue sua URL_WORKER.
  2. Adicione /healthz no final.
  3. Abra essa URL no navegador.
Exemplo do formato correto 
https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/healthz
Resposta esperada no navegador 
{"ok":true,"service":"jurisprudenciaia-mcp"}

Se a resposta acima aparecer, o Worker está online. Se der erro 404, 500 ou página em branco, volte ao passo 6 e abra o log do controlador de deploy.

8

Conecte o MCP no cliente

Use a URL com /mcp no final. Escolha um caminho: Claude.ai e ChatGPT usam OAuth; Codex usa Bearer por variável local.

Escolha onde você vai usar o conector. O guia mostra apenas os campos daquele cliente para evitar mistura de tokens.

Codex

Use esta tela quando o conector for usado no Codex Desktop.

  1. No Worker, grave o secret MCP_BEARER_TOKEN.
  2. No computador, crie a variável local MCP_BEARER_TOKEN com o mesmo valor.
  3. Feche e abra o Codex depois de criar ou alterar a variável.
  4. No Codex, abra Conectar a um MCP personalizado.
  5. Preencha os campos abaixo e rode o teste antes de procurar erro na interface.
Campos da tela do Codex 
Nome: jurisprudenciaia-mcp
Tipo: HTTP com streaming
URL: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp
Variável de ambiente de token do portador: MCP_BEARER_TOKEN
Windows: variável de usuário 
[Environment]::SetEnvironmentVariable("MCP_BEARER_TOKEN", "<mesmo valor do Worker>", "User")
macOS: Codex Desktop 
launchctl setenv MCP_BEARER_TOKEN "<mesmo valor do Worker>"
macOS: Terminal ou CLI 
echo 'export MCP_BEARER_TOKEN="<mesmo valor do Worker>"' >> ~/.zshrc
source ~/.zshrc
Teste no Windows 
$env:MCP_BEARER_TOKEN = [Environment]::GetEnvironmentVariable("MCP_BEARER_TOKEN", "User")
npm run check:codex-http -- https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp
Teste no macOS 
export MCP_BEARER_TOKEN="$(launchctl getenv MCP_BEARER_TOKEN)"
npm run check:codex-http -- https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp

Teste correto: Conectado, Ferramentas encontradas (5) e Chamada real OK.

No macOS, launchctl setenv é o caminho para o Codex Desktop aberto pelo Dock, Finder ou Spotlight. Se reiniciar o Mac e a variável sumir, rode o comando novamente antes de abrir o Codex.

ChatGPT

Use esta tela para criar um app em modo desenvolvedor no ChatGPT. Não use o token do Codex aqui.

  1. Abra Configurações, depois Aplicativos.
  2. Ative o Modo desenvolvedor e clique em criar novo app.
  3. Em Conexão, escolha URL do servidor e cole a URL /mcp.
  4. Em Autenticação, escolha OAuth.
  5. Em Método de registro, escolha Cliente OAuth definido pelo usuário.
  6. Informe exatamente o mesmo Client ID e Client Secret salvos no Worker.
  7. Use client_secret_post como método de autenticação do endpoint de token.
  8. Ignore os avisos de DCR e CIMD indisponíveis; eles são esperados neste modo.
Campos principais do ChatGPT 
Nome: jurisprudenciaia-mcp
Conexão: URL do servidor
URL: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp
Autenticação: OAuth
Método de registro: Cliente OAuth definido pelo usuário
OAuth Client ID: jurisprudenciaia-mcp-client
OAuth Client Secret: <mesmo valor do Worker Secret MCP_OAUTH_CLIENT_SECRET>
Método de autenticação do endpoint de token: client_secret_post
Endpoints avançados do ChatGPT 
URL de autorização: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/oauth/authorize
Token URL: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/oauth/token
URL de registro: deixe em branco
Endereço base do servidor de autorização: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev
Recurso: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp
Escopo, se o ChatGPT pedir: jurisprudenciaia:search
Teste antes de criar o app 
$env:MCP_OAUTH_CLIENT_ID = Read-Host "Cole o OAuth Client ID do Worker"
$env:MCP_OAUTH_CLIENT_SECRET = Read-Host "Cole o OAuth Client Secret do Worker"
npm run check:chatgpt-oauth -- https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp

Se aparecer erro genérico ao conectar, o caso mais comum é Client ID ou Client Secret diferente do Worker. Mantenha o Worker secret MCP_OAUTH_CLIENT_ID com o valor jurisprudenciaia-mcp-client.

Claude.ai

Use esta tela quando o conector for usado dentro do Claude.ai.

  1. Entre em claude.ai.
  2. Abra Settings e depois Connectors.
  3. Adicione um conector personalizado e cole a URL /mcp.
  4. Abra Advanced settings, se aparecer.
  5. Informe o Client ID e o Client Secret salvos no Worker.
  6. Salve e autorize a conexão quando o Claude.ai pedir.
Campos do Claude.ai 
URL: https://jurisprudenciaia-mcp.<seu-subdominio>.workers.dev/mcp
OAuth Client ID: jurisprudenciaia-mcp-client
OAuth Client Secret: <mesmo valor do Worker Secret MCP_OAUTH_CLIENT_SECRET>

O Claude.ai costuma descobrir os endpoints OAuth automaticamente pela URL /mcp. Se ele pedir campos avançados, use os mesmos endpoints do ChatGPT.

Não cole MCP_ACCESS_TOKEN_SECRET em nenhum cliente. No Claude.ai e no ChatGPT use MCP_OAUTH_CLIENT_ID e MCP_OAUTH_CLIENT_SECRET. No Codex use MCP_BEARER_TOKEN.

9

Faça o primeiro teste no cliente escolhido

Agora confirme que as ferramentas aparecem e que a consulta retorna Markdown.

  1. Abra uma conversa nova no Claude.ai, ChatGPT ou Codex.
  2. Cole a mensagem abaixo.
  3. Confirme que o cliente usa a ferramenta do MCP.
  4. Confira se o resultado contém resposta, tese principal, precedentes e cautelas.
Mensagem de teste 
Use consultar_jurisprudenciaia para pesquisar responsabilidade civil por negativação indevida dano moral.
Ferramentas esperadas 
consultar_jurisprudenciaia
pesquisar_jurisprudencia
buscar_precedentes
analisar_tese_juridica
comparar_teses_juridicas

Resultado correto: a resposta deve começar com # Resultado JurisprudenciaIA e conter ## Precedentes citados.

Se algo der errado

  • O controlador de build falhou antes do deploy. Abra o log. Se aparecer erro de secrets ausentes, volte ao passo 5 e confira os nomes obrigatórios no Worker.
  • Cloudflare recusou o deploy. Confira se o token tem permissão para editar Workers e se o Account ID pertence à mesma conta.
  • /healthz não abre. Confira se você copiou a URL do Worker, não a URL do repositório GitHub.
  • O cliente mostra erro de autenticação. No Claude.ai ou ChatGPT, confira Client ID e Client Secret. No Codex, rode npm run check:codex-http e confira se MCP_BEARER_TOKEN existe no Worker e como variável local no Windows ou no macOS.
  • Codex não conecta no HTTP com streaming. Confira se a opção HTTP com streaming está selecionada, se a URL termina em /mcp, se o campo de token do portador aponta para MCP_BEARER_TOKEN e se o Codex foi fechado e aberto depois da variável local ser criada.
  • O cliente mostra apenas uma ferramenta ou schema antigo. Desconecte o conector, conecte novamente e refaça o teste da etapa 9.
